Nodejs spider

发表于 2018-10-20

之前也写过Nodejs爬虫项目，当时初学Nodejs，做出的项目感觉不系统，后来又重构了phantomjs版本，以及目前的puppeteer版本，这里简单介绍下涉及的技术点同时贴出关键代码，项目地址

目标网站 https://www.dmzj.com/info/blame.html

预期目的 把一部漫画的所有图片分章节下载到本地

我们先看下Nodejs 爬虫常用的一些模块或工具：

Puppeteer:
对chromium做的接口封装，而且目前已公布要支持firefox内核(短期来看击败其他同类模块没悬念)
抓取内容时可以解析JS 适用于动态渲染页面 支持ES6
调试十分便捷
Google维护
其他功能：
​ 利用网页生成PDF、图片
​ 爬取SPA应用，并生成预渲染内容（即服务端渲染）
​ 自动化表单提交、UI测试、键盘输入等
​ 自动化测试

PhantomJS:
headless browser webkit内核
抓取内容时可以解析JS 适用于动态渲染页面 但不支持ES6
调试比较黑盒
API丰富 目前缺少维护
有对应的Nodejs环境模块 phantomjs-node

cheerio：
jQuery Core 子集
不能渲染css以及js
适用于静态页面以及字符串模版的情况

async：
异步流程控制模块
包括常用的串行和并行

http/https：
网络请求模块
同类型库 request

fs：
文件系统模块，用于操作本地文件

网页结构研究完毕，可以正式开始写代码了

先看下文件结构

lib里存放章节页面和分页面对应的爬虫，imgs文件夹存放请求下来的图片，bin是commander.js需要的入口文件，commander.js用于命令行工具的开发，后续可能会写篇文章单独来分享，这里暂不展开。

我们先来看章节页面爬虫代码

首先我们生成了puppeteer实例，让它打开chromium的headless模式，等待页面渲染成功之后开始解析DOM，获取分章节的跳转链接、章节名称，之后开始在imgs文件夹下生成以章节名称命名的文件夹，之后用asyncModule.series方法串行管理每个章节图片的下载任务，这里不用并行主要是因为容易被目标网站封禁IP，当然这里也可以分出其他线程来执行任务。
之后我们来看子页面爬虫的代码

和章节页面爬虫的区别主要在于，启用了请求拦截器，由于网络问题，有某个链接的资源请求会超时，导致页面加载很慢，这时我们通过用请求拦截的方式直接禁止这个请求的发起。请求拦截器还可以用于拦截图片请求，让页面加载更加迅速。在获取所有图片地址后，我们通过asyncModule.mapLimit进行并发请求，同时可以限制并发数量。
